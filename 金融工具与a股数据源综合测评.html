交叉分析

核心发现1：AKShare一拖三满足所有A股需求

AKShare是当前A股开源数据源的"事实标准"：

  • GitHub 8000+ star，持续更新超过5年
  • 覆盖A股/期货/基金/债券/宏观全品类
  • 数据来源：东方财富+新浪财经+同花顺等多源融合
  • Vibe-Trading内置AKShare作为数据源，无需额外配置

唯一短板：实时接口延迟约500ms，不适合高频交易。但对于你做策略回测、趋势判断、量价分析来说完全够用。

核心发现2：Vibe-Trading是唯一能把这些数据用起来的工具

Vibe-Trading是HKUDS系列里唯一面向终端用户的工具，不同于AI-Trader（比赛平台）和TradingAgents（研究框架）：

  • 自然语言驱动：你说"写一个MACD金叉回测" → 自动写代码+跑回测+出报告
  • 71个金融技能：策略生成/因子分析/回测/跨市场组合/风控
  • 29个专家团队：投资委员会/量化策略组/风控组等
  • 7大回测引擎：A股/港股/美股/加密货币/期货/外汇全覆盖
  • 一键导出到通达信/同花顺格式
  • 自带MCP Server可直接集成到Hermes

AKShare和Tushare Pro是唯二值得考虑的。efinance/qstock/adata社区太小，数据源单一（大多也只爬东方财富），出问题没人解决。

核心发现4：MCP数据源 vs 工具内置数据源

你当前在Hermes上，理论上Tushare Pro MCP Server可以直配到config.yaml，让Hermes直接调A股数据。但这不如用Vibe-Trading，因为：

  • Vibe-Trading自带AKShare，天然支持A股
  • 你不需要额外装Tushare Pro MCP + 申请Token + 管理积分
  • Vibe-Trading还有回测引擎、策略生成、多Agent团队等能力
  • Vibe-Trading本身也暴露MCP工具给Hermes，一箭双雕

综合判断

装一个Vibe-Trading就够了。 它自带A股数据源（AKShare），自然语言驱动，pip install 5分钟，能回测能分析能导出，还剩了你去配Tushare Token/积分/频控的麻烦。其他工具和数据源要么不支持A股，要么需要你装Claude生态，要么社区太小没人维护。

一步行动建议：pip install vibe-trading-ai，跑一个A股回测验证效果。验证通过后，把Hermes的MCP配置指向Vibe-Trading，日常A股分析直接在Hermes里完成。

风险提示

  • AKShare的实时数据有约500ms延迟，不影响回测/趋势判断，影响高频交易
  • Tushare免费版有频率限制（100次/分钟），需注意调用节奏
  • Vibe-Trading是2026年4月新项目，仍在快速迭代中，可能有API变化
  • 所有AI生成的策略策略回测结果和实盘表现可能有差异
